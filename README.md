## 무엇을 보여주나

같은 소재 15종을 두 가지 조건으로 집행한 결과를 담습니다.

| | 1차 | 2차 |
|---|---|---|
| 기간 | 2026.7.17 – 7.20 (종료) | 2026.7.20 – (진행 중) |
| 타겟 연령 | 성인 18세 이상 | **13세 이상** (구버전 수동 캠페인) |
| 성과 목표 | 랜딩 페이지 조회 | 링크 클릭 |
| 예산 | 소재당 일 ₩10,000 | 소재당 일 ₩10,000 |

### 탭 구성

- **1차 · 성인 18+** — KPI 요약, 지표(결과/CPA/지출/노출) × 연령 필터, 소재 랭킹(탭하면 연령별 상세 + 소재 이미지), 소재×연령 히트맵, 인사이트, 원데이터
- **2차 · 13세+** — 동일 구조에 **13-17 연령 필터 포함**. 10대 노출·클릭이 소재별로 어디서 나오는지 확인 가능
- **비교 · 합산** — 소재별 1차 CPA vs 2차 CPC 나란히, 비용 효율 순위 변동(▲▼), CPM 변화, 합산 KPI

> 1차와 2차는 성과 지표가 다르므로(랜딩 조회 vs 링크 클릭) 절대값 비교 대신 **차수 내 순위**와 **CPM**으로 비교하도록 설계했습니다.

## 데이터

- 출처: Meta 광고 관리자 → 연령 분석 데이터 적용 → CSV 내보내기
- 데이터는 `index.html` 안에 정적으로 내장되어 있습니다 (외부 API 호출 없음)
- 광고차단 사용자 등으로 소폭 과소집계될 수 있어, 소재 간 **상대 비교** 용도로 권장

### 갱신 방법

1. 광고 관리자에서 기간 설정 + 분석 데이터(연령) 적용 후 CSV 내보내기
2. `index.html`의 `ROWS`(1차) / `P2ROWS`(2차) 배열을 새 수치로 교체
3. `main`에 push → GitHub Pages가 자동 재배포 (링크 동일)

## 기술

- 정적 단일 파일 (`index.html`) — 프레임워크·차트 라이브러리·외부 의존성 **0**
- 순수 CSS/JS로 구현한 바 차트·히트맵·툴팁, 색약 안전 검증을 거친 연령 팔레트
- 소재 썸네일은 `img/`에 360px JPEG로 최적화 (장당 약 25KB)
- 호스팅: GitHub Pages (`main` push = 배포)
